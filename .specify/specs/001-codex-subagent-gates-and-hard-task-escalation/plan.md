# Plan: Codex subagent gates and hard-task escalation

> The implementation strategy for `spec.md`. Answers **how**, not what.

## 1. Approach

Extend `build-adapters.sh` — the one script that already owns every Codex
artifact — with a fourth generation block: three subagent TOMLs under
`~/.codex/agents/`, built from the canonical persona files plus the model
policy, with marker-based pruning copied from the existing tier-profile block
(REQ-002). Split the shared single-agent `PREAMBLE` into a Codex variant that
instructs delegation (REQ-003) and a Copilot variant that keeps today's
persona-pass wording (REQ-006 gate). Correct the stale claims in the four doc
surfaces (REQ-004), widen the codex-effort whitelist in the two validation
sites (REQ-005), and close with two captured empirical runs — codex accepts
and can spawn the TOMLs (AC-007), copilot handoff works or doesn't (AC-008).

## 3. Data model

New generated artifacts (machine-local, never committed):
`~/.codex/agents/sdd-{opponent,reality-check,implement-hard}.toml` —

```toml
# Generated by sdd-kit (build-adapters.sh) from models.yml — do not edit.
name = "sdd-opponent"
description = "<persona frontmatter description>"
model = "<codex_model of the role's tier>"            # omitted when unset
model_reasoning_effort = "<codex_effort of the tier>" # omitted when unset
developer_instructions = '''
<full persona markdown body>
'''
```

The first-line marker comment is the ownership test (CON-003): prune only
files that carry it. Persona bodies go in TOML multi-line **literal** strings
(`'''`) — no escape processing, so persona backslashes (e.g.
`reality-?check.*\.md`) survive verbatim; neither persona contains `'''`
(verified this session). Guard anyway: if a persona body ever contains `'''`,
fail the build with a clear error rather than emit broken TOML.

## 4. API / contracts

No public API. The kit-generated Codex artifact set grows from
`sdd-<tier>.config.toml` (profiles) to profiles + `agents/sdd-*.toml`.

### Internal seams (pre-decided)

- `build-adapters.sh` new function `codex_subagent_toml()` — usage:
  `codex_subagent_toml <name> <role> <description> <instructions-file>`
  writes to stdout; caller redirects to
  `$HOME/.codex/agents/<name>.toml`. `<instructions-file>` is the persona path
  for gates; for implement-hard the caller writes a heredoc to a temp file
  first. Model/effort resolved inside via the existing `policy()` helper
  (`policy <role> codex model|effort`).
- Subagent block placement: after the existing tier-profile block, same
  `[[ -d "$HOME/.codex" ]] && (( HAVE_POLICY ))` guard, its own `keep=`/prune
  loop over `"$HOME"/.codex/agents/sdd-*.toml` testing the marker via
  `head -1` + `grep -F 'Generated by sdd-kit'`.
- Kit marker string, verbatim (shared with profiles):
  `# Generated by sdd-kit (build-adapters.sh) from models.yml — do not edit.`
- `CODEX_PREAMBLE` delegation wording (transcribe): gates — "delegate the gate
  to the kit subagent `sdd-opponent` / `sdd-reality-check`
  (`~/.codex/agents/`, generated by the SDD kit); it reviews with fresh
  context. If the subagent is not installed, fall back to the persona-pass
  below and over-correct toward suspicion."; escalation — "a task marked
  `[hard]`, any failed-acceptance retry, and every gate follow-up
  (`T###o*`/`T###a*`) is delegated to `sdd-implement-hard`."
- `implement-hard` developer_instructions (inline heredoc, ~8 lines): work in
  the caller-named worktree; implement exactly the quoted task; transcribe the
  plan's seams; run the task's *Verify:* command and return files changed +
  the command + its output verbatim.
- Effort whitelist regex, both sites in `model-policy.sh`:
  `none|minimal|low|medium|high|xhigh|ultra|max` (error text updated to
  match).
- Test hermeticity contract: `tests/test-build-adapters.sh` copies the kit
  (`git ls-files` set) into `$SANDBOX/kit`, writes `$SANDBOX/kit/models.yml`,
  and runs with `HOME="$SANDBOX/home"` — the real `~/.codex`, `~/.copilot`,
  and repo `models.yml` are never read or written.

## 7. Risks

- **R1:** Installed codex (0.144.1) rejects or ignores the TOML shape the docs
  describe (docs may lag/lead the binary) — *Mitigation:* AC-007 is an
  empirical task early in the Tests stage: generate, then `codex exec`
  headlessly and capture; if the shape is wrong, fix the generator against
  observed behavior before the docs tasks land.
- **R2:** Persona text breaks TOML parsing — *Mitigation:* multi-line literal
  strings + the `'''` build guard (§3); AC-007 exercises real parsing.
- **R3:** Copilot handoff doesn't work headlessly and time sinks —
  *Mitigation:* AC-008 is explicitly allowed to conclude "doesn't work";
  timebox to one scratch-dir experiment; docs stay persona-pass either way.
- **R4:** Pruning deletes a user's own agent TOML — *Mitigation:* marker test
  (CON-003) + AC-002's user-file fixture proves non-deletion.

## 8. Rollout

- Ships as one PR on `spec/001-…` branch; merge → port to private repo
  (CON-004) → `setup.sh` on the live install regenerates `~/.codex/agents/`.
- No flag: generation is gated on `~/.codex` existing + models.yml present,
  identical to today's profile behavior; absence of the TOMLs falls back to
  persona-pass (REQ-001), so a half-rolled-out state is safe.
- Observability: MET-001 is checked at the next Codex-dispatched spec's retro
  (gate report provenance); no runtime telemetry exists to wire.
- Reversible: delete `~/.codex/agents/sdd-*.toml` (or re-run with the block
  reverted) — persona-pass fallback resumes.
